ERROR

Durante el proceso de precargue de operadores en prefacturación se puede presentar en RR, el error mostrado en la imagen.

CAUSAS

Este error se puede presentar por varias causas, dentro de las cuales se tienen documentadas las siguientes:

Porque la tarifa donde se presenta la novedad tiene un servicio de telefonía, pero no tiene cargado ni paquetes ni funcionalidades

Que el cliente no se encuentre activo

El número de teléfono se encuentra desconectado o no corresponde al del cliente

Que el cliente fue movido por el área de HOME PAS

SOLUCIÓN

Para validar que las funcionalidades y paquetes se encuentren en la tarifa:

Se ingresa en la opción 7 – 1. Mantener Tarifas y Servicios, se ingresa a la tarifa de las cuentas afectadas, en su última versión

Una vez dentro de los servicios de la tarifa se oprime la tecla F8 y seguido la tecla F9. Allí se debe validar que se encuentren todas las funcionalidades o por lo menos la reportada en el mensaje de error. Se sale de la opción luego de revisar oprimiendo F3

Luego se ingresa a validar que las tarifas de las cuentas afectadas tengan el paquete FUN. Para eso, dentro de la tarifa se ingresa con F8 – luego nuevamente F8 - F8 - Shif+F6. Allí se debe observar que se encuentre el paquete mencionado, como se observa en la imagen.

COPIADO DE FUNCIONES Y PAQUETES

En caso de que las tarifas no tengan las funcionalidades o los paquetes no estén, deben copiarse desde una tarifa que ya los tenga. Para hacer esta copia, la tarifa donde se van a copiar estos parámetros debe tener fecha futura mayor al día presente, de no ser así arrojara el siguiente error: “You can't copy packages or feature sets to current rate schedules.” y solo se pueden copiar a las comunidades ***. Una vez validado esto los pasos a seguir son los siguientes:

Posteriormente se toma una tarifa de referencia que tenga tanto las funcionalidades como los paquetes requeridos. Para este ejemplo escogimos como referencia la tarifa PLZC.

Tener presente que tanto la tarifa de referencia como en la que se va a copiar deben corresponder a la misma división y por el momento aplica solo para división ***

Luego de esto se ingresa oprimiendo 2 veces la tecla F8. Se copian todos los datos de la columna resaltada en el mismo espacio, pero de la tarifa en donde se va a copiar la funcionalidad (la PPP para este ejemplo)

ANTES DESPUÉS

se regresa a la pantalla principal de la tarifa, se sigue la ruta para llegar a los paquetes (F8 - F8 - Shif+F6), allí con la opción 3 copiamos el paquete que deseamos en la tarifa que lo requiere

Esto no lleva a una pantalla en donde se deben ingresar los datos de la tarifa a donde se va a copiar la funcionalidad, se da enter y listo

ANTES DESPUÉS

Para el copiado de las funcionalidades el proceso es el mismo solo que este se realiza en la opción F8 y seguido la tecla F9.

Para validar si el cliente se encuentra activo y si la línea corresponde, se ingresa en la opción 1. búsqueda o Consulta del Suscriptor o la Casa. Allí se registra el numero de cuenta del usuario y se oprime F7

En la parte superior derecha se observa el estado del usuario, de estar inactivo, se debe reportar y alli terminaría l atención de la novedad

En caso de que el usuario este activo, se ingresa al listado de las ordenes de trabajo, allí se escoge la última orden de trabajo con Shif + F7,

Una ves dentro de esta se oprime Shif + 3 para ver el número de teléfono del usuario y allí validamos con 1, si el número de teléfono que este tiene,coincide con el reportado en la novedad

Por último, se valida que el cliente no haya sido movido por el área de HOME PAS. Esto se valida revisando en que división y comunidad se han generado normalmente las ordenes de trabajo, esto se valida en la opción

1. Búsqueda o Consulta del Suscriptor o la Casa. Allí se coloca el numero de cuenta, se oprime F7, luego shift + F7

En el paso anterior se valido a que divisiones se le realizo ordenes de trabajo, ahora se valida a que divisiones se le esta cobrando, si estas son diferentes, se escala al área encargada. Para ver a que división y comunidad se le esta realizando el cobro, luego de haber ingresado a la cuenta. Se digita shift+F1 luego se oprime shift+F10 para ver el histórico de facturas

Si las divisiones y comunidades no coinciden con las de las ordenes de trabajo, quiere decir que ese cliente fue movido de división y/o comunidad y por eso se presenta la falla , como se observa en la siguiente imagen.

Una vez terminadas las validaciones indicadas, la falla persiste, se debe dirigir al área de Soporte
