ERROR POR AUTORIZACION EN TABLA FV - CODIGO (SXTITL) – SI EL CAMPO TITULO (TITLE) ESTA BLOQUEADO EN EL MENU DE SUSCRIPTORES

Cuando el campo “TITULO (TITLE)” no permite realizar ninguna modificación, es decir aparece bloqueado, el error se puede presentar por dos motivos:

El usuario debe tener autorización sobre el objeto que corresponde a ese campo, el cual se debe solicitar al área de accesos por medio de un caso.

Si usuario ya tiene autorización sobre el objeto, el bloqueo sobre el campo se debe a que no tiene autorización (Permisos) sobre la tabla “FV”, código “SXTITL - SUBSCRIBER'S TITLE”, por tal motivo debe generar un caso al área de parametrización RR, por la rama de “Modificar parámetros de constantes” con el formato firmado por el líder o gerente de área y el/los usuarios que requieren los permisos, cabe recalcar que la pestaña a diligenciar es la de permisos.

El error se puede evidenciar y corregir bajo la siguiente ruta:

Menú principal de RR

Opción “1. Busqueda o Consulta del Suscriptor o la Casa”

Cuenta a modificar

Se accede a la opción F21=ActSus (Shift + F9)

Redirecciona a la siguiente ventana, en donde aparece el campo “TITULO (TITLE)”, el cual aparece bloqueado (Como se evidencia en la imagen el campo no permite realizar ninguna modificación, ya que todo está bloqueado y en color verde) para poder activar o realizar alguna modificación sobre ese campo se debe tener permisos o autorización sobre la tabla “FV” – código “SXTITL”.

Se va a la opción menú principal, luego opción “7. Parametros del Sistema” y después “4. Mantener Tablas del Sistema” / Tabla “FV” / Código “SXTITL” / por ultimo “8=Authority” para colocar el/los usuarios a autorizar sobre el campo.

Ejemplo de demostración, se coloca un usuario que no tenga el permiso

Se evidencia que ya le aparece habilitado el campo TITULO (Color azul) y puede realizar modificaciones.
